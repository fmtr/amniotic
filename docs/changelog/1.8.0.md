- Migrated core Amniotic models and controls away from dataclass field metadata to Pydantic `Field` usage for excluded/internal attributes, including theme/control/device/recording objects.
- Added process and object monitoring in the device lifecycle (`monitor_objects_task`) with periodic memory/thread/object-class debug telemetry via `psutil`, and started this monitor during device initialization.
- Hardened background metadata refresh handling by splitting refresh task logic, improving loop error isolation, and adding exception logging around monitoring/transcoding/download paths.
- Updated recording/theme object construction flows (including explicit device/path wiring for recording instances) and added additional debug instrumentation for stream initialization and chunk/transcoding iteration.
- Expanded dependency and packaging metadata: added `psutil` runtime dependency, added `pytest`/`pytest-asyncio` test dependencies, and introduced `pyproject.toml` `project` metadata plus local `uv` source mappings for `corio` and `haco`.
- Added substantial regression test coverage for client startup/supervisor MQTT discovery, device media-player state parsing/replay, and MQTT control announce/command behavior, with new HA/supervisor fixture payloads.
- Revised native install docs to reflect Pydantic-based configuration options (config file, environment variables, CLI flags), corrected command examples, and removed unnecessary audio stack package requirements from install steps.
